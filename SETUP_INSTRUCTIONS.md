# MkDocs Search Integration - Setup Instructions

This implementation uses **lunr.js** (the same library MkDocs uses) to search the Quix documentation directly in your Angular app.

## Installation

### 1. Install Dependencies

```bash
npm install lunr @types/lunr
```

### 2. Add Files to Your Angular Project

Copy the following files to your Angular project:

```
src/
  app/
    services/
      mkdocs-search.service.ts    # Service that loads and searches MkDocs index
    components/
      doc-search/
        doc-search.component.ts    # Search component
        doc-search.component.html  # Template
        doc-search.component.scss  # Styles
```

### 3. Update Your Module

Add the component and service to your Angular module:

```typescript
// app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { HttpClientModule } from '@angular/common/http';
import { ReactiveFormsModule } from '@angular/forms';

import { AppComponent } from './app.component';
import { DocSearchComponent } from './components/doc-search/doc-search.component';
import { MkDocsSearchService } from './services/mkdocs-search.service';

@NgModule({
  declarations: [
    AppComponent,
    DocSearchComponent
  ],
  imports: [
    BrowserModule,
    HttpClientModule,
    ReactiveFormsModule
  ],
  providers: [MkDocsSearchService],
  bootstrap: [AppComponent]
})
export class AppModule { }
```

### 4. Use the Component

Add the search component to any template:

```html
<!-- In your app.component.html or any other template -->
<app-doc-search></app-doc-search>
```

## How It Works

### Architecture

```
┌─────────────────────────────────────────────────────────┐
│  Angular App                                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │  DocSearchComponent                             │   │
│  │  - User types search query                      │   │
│  │  - Displays results                             │   │
│  └──────────────┬──────────────────────────────────┘   │
│                 │                                       │
│                 ▼                                       │
│  ┌─────────────────────────────────────────────────┐   │
│  │  MkDocsSearchService                            │   │
│  │  - Loads search_index.json from MkDocs          │   │
│  │  - Builds lunr.js index                         │   │
│  │  - Searches using lunr.js                       │   │
│  └──────────────┬──────────────────────────────────┘   │
│                 │                                       │
└─────────────────┼───────────────────────────────────────┘
                  │
                  │ HTTP GET
                  ▼
┌─────────────────────────────────────────────────────────┐
│  https://quix.io/docs/search/search_index.json         │
│  - Generated by MkDocs build                           │
│  - Contains all searchable content                     │
│  - Updated when docs are rebuilt                       │
└─────────────────────────────────────────────────────────┘
```

### Search Flow

1. **On App Load**: Service fetches `search_index.json` from the MkDocs site
2. **Index Building**: lunr.js builds a search index from the documents (same as MkDocs does)
3. **User Types**: Input is debounced (300ms) to avoid excessive searching
4. **Search Execution**: lunr.js searches the index using the same algorithm as MkDocs
5. **Results Display**: Matching documents are shown with highlighted search terms
6. **Click to Open**: Results open the documentation page in a new tab

## Configuration

### Customize Search Index URL

If your MkDocs documentation is hosted at a different URL, update the service:

```typescript
// mkdocs-search.service.ts
private readonly SEARCH_INDEX_URL = 'https://YOUR-DOMAIN.com/docs/search/search_index.json';
private readonly DOCS_BASE_URL = 'https://YOUR-DOMAIN.com/docs/';
```

### Adjust Result Limit

Change the number of results returned:

```typescript
// doc-search.component.ts
this.searchService.search(query, 20) // Default is 15
```

### Modify Debounce Time

Adjust how long to wait after user stops typing:

```typescript
// doc-search.component.ts
debounceTime(500) // Default is 300ms
```

## Troubleshooting

### CORS Issues

If you get CORS errors, the MkDocs site might not allow cross-origin requests. Solutions:

1. **Use a proxy** in your Angular dev environment:
   ```json
   // proxy.conf.json
   {
     "/docs": {
       "target": "https://quix.io",
       "secure": true,
       "changeOrigin": true
     }
   }
   ```

   Then update the service URL:
   ```typescript
   private readonly SEARCH_INDEX_URL = '/docs/search/search_index.json';
   ```

2. **Download and host locally**: Copy `search_index.json` to your assets folder

### Index Won't Load

Check the browser console for errors. Common issues:

- URL is incorrect
- MkDocs site is down
- CORS blocking the request
- JSON file is malformed

### No Results Found

- Ensure index is ready (check "X documents indexed")
- Try simpler search terms
- Check browser console for lunr.js errors
- Verify the search_index.json loaded correctly

## Advanced Usage

### Add Caching

Cache the search index in localStorage:

```typescript
// In mkdocs-search.service.ts
private loadSearchIndex(): void {
  // Try to load from cache first
  const cached = localStorage.getItem('search_index');
  const cacheTime = localStorage.getItem('search_index_time');

  if (cached && cacheTime) {
    const age = Date.now() - parseInt(cacheTime);
    if (age < 24 * 60 * 60 * 1000) { // 24 hours
      const data = JSON.parse(cached);
      this.documents = data.docs;
      this.buildLunrIndex(data);
      this.indexReady$.next(true);
      return;
    }
  }

  // Load from network
  this.http.get<MkDocsSearchIndex>(this.SEARCH_INDEX_URL)
    .subscribe(data => {
      localStorage.setItem('search_index', JSON.stringify(data));
      localStorage.setItem('search_index_time', Date.now().toString());
      // ... rest of code
    });
}
```

### Filter by Section

Add section filtering:

```typescript
// In mkdocs-search.service.ts
searchInSection(query: string, section: string, limit: number = 10): Observable<SearchResult[]> {
  return this.search(query, limit * 2).pipe(
    map(results => results.filter(r => r.location.startsWith(section)).slice(0, limit))
  );
}
```

### Track Recent Searches

Save user's recent searches:

```typescript
// In doc-search.component.ts
private saveRecentSearch(query: string): void {
  const recent = JSON.parse(localStorage.getItem('recent_searches') || '[]');
  if (!recent.includes(query)) {
    recent.unshift(query);
    localStorage.setItem('recent_searches', JSON.stringify(recent.slice(0, 10)));
  }
}
```

## Comparison with Custom Ingestion

| Feature | MkDocs lunr.js | Custom Vector Search |
|---------|---------------|---------------------|
| Setup | Simple - just install lunr | Complex - Python, Qdrant, embeddings |
| Maintenance | Auto-syncs with MkDocs | Manual ingestion required |
| Infrastructure | Static JSON file | Vector DB + API server |
| Search Type | Keyword-based | Semantic (AI-powered) |
| Cost | Free | Hosting costs |
| Latency | Instant (client-side) | Network + processing time |
| Offline | Yes (can cache) | No |
| Quality | Good for exact matches | Better for conceptual queries |
